## Admin Dashboard Forms
All forms integrate with Neon Postgres via Drizzle ORM and enforce locale-aware content. Unless noted, fields are required. Provide inline validation, autosave drafts, and change history where feasible.

### Common Form Patterns
- **Locale Tabs:** EN and ES inputs side-by-side with auto-translation suggestions from Agent Zero.
- **Slug & Metadata:** Auto-generate URL slugs; allow manual override with validation (lowercase, hyphenated, < 80 characters).
- **Media Picker:** Connect to S3/R2 buckets with thumbnail previews and enforced alt text per locale.
- **Status Controls:** Draft, Scheduled, Published states with audit log entries.

### 1. Content (Stories, Tutorials, Media Collections)
- **Stories/Tutorials Form Fields:**
  - Title (EN, ES)
  - Summary (EN, ES; max 160 characters for SEO)
  - Body (EN, ES; rich text with block quotes, media embeds)
  - Cover Media (file selector → `media.id`)
  - Tags (multi-select; e.g., GeoAg, Curriculum, Youth Voices)
  - Publish Date (with timezone awareness)
  - Featured Toggle (boolean)
  - SEO Overrides (meta title, description, OG image)
  - Accessibility Checklist (checkbox confirmation for captions, contrast, transcripts)
- **Tutorial Steps Subform:** Repeatable entries with Step Title (EN/ES), Step Body (EN/ES), Reference Media, Estimated Time, Materials List (structured array).
- **Version Notes:** Text area for editor change log; visible in audit trail.

### 2. Media Library
- **Media Upload Fields:**
  - File Upload (image, video, PDF; limit 500 MB)
  - Title (EN, ES)
  - Alt Text (EN, ES)
  - Description / Caption (EN, ES)
  - Credit Line (text)
  - Usage Rights (dropdown: Owned, Creative Commons, Contributor License)
  - Tags (multi-select: Youth, Tower Build, Diagram, Sponsor, etc.)
  - Source URL (optional)
  - Consent Documentation (file or checkbox link for youth media releases)
- **Generated Metadata:**
  - MIME type, file size, dimensions/duration, upload user, checksum
  - AI Alt Text Suggestions stored as JSON for review

### 3. Donors
- **Donor Profile Fields:**
  - Donor Type (Individual, Organization, Anonymous)
  - First Name, Last Name (store hashed ID for public display)
  - Display Preference (Full name, Initials only, Hidden)
  - Email (with consent checkbox for updates)
  - Phone (optional, E.164 format)
  - Locale Preference (EN, ES)
  - Address (structured: country, state, city, postal code)
  - Consent Flags (newsletter opt-in, public recognition, impact reports)
  - Notes (internal)
- **Donations Subtable:** Auto-linked records showing amount (cents), currency, campaign, payment source (Stripe, Giving Block), transaction reference, receipt status, tax acknowledgment sent (boolean), opt-out of realtime display (boolean).
- **Receipting Actions:** Buttons to resend receipt, download PDF, export CSV selection.

### 4. Campaigns
- **Campaign Form Fields:**
  - Title (EN, ES)
  - Short Description (EN, ES)
  - Hero Media (image/video)
  - Goal Amount (cents) & Currency
  - Start Date, End Date (with optional open-ended toggle)
  - Impact Conversions (repeatable: label EN/ES, amount, units—e.g., Tower = $100)
  - Progress Visibility Settings (public/private, show donor brackets)
  - Featured on Homepage (boolean)
  - Share Copy (EN, ES; used for meta + share modal)
  - Webhook Keys / IDs for Stripe & Giving Block integration
  - Auto-close conditions (optional, e.g., close when goal reached + 10%)

### 5. Donation Webhooks & Realtime Settings
- **Configuration Fields:**
  - Stripe Endpoint Secret, Stripe Account ID
  - Giving Block API Key, Campaign ID
  - Upstash Redis URL, Pub/Sub Token
  - Polling Interval (seconds) for fallback SWR
  - Data Privacy Rules (dropdown: Show bracket, Show initials, Hide amount for minors)
  - Test Mode Toggle (routes events to staging branch in Neon)
  - Alert Emails/Slack Webhook for failures

### 6. Newsletter
- **Subscriber Management Fields:**
  - Email, Locale (EN/ES)
  - Source (Homepage, Event QR, Manual Import)
  - Status (Subscribed, Unconfirmed, Unsubscribed)
  - Tags/Segments (Educator, Donor, Student, Sponsor)
  - Consent timestamp (ISO)
- **Campaign Log Form:**
  - Subject (EN, ES)
  - Body (markdown support)
  - Audience Segment selector
  - Scheduled Send Time (UTC)
  - Buttondown/Mailchimp Template ID
  - Post-send metrics (opens, clicks) ingested via webhook

### 7. Agents (Agent Zero Task Console)
- **Task Creation Fields:**
  - Prompt Title
  - Prompt Body (markdown)
  - Locale Output Request (EN, ES, Bilingual)
  - Task Type (Content draft, Translation, Donor email, Impact summary, QR refill card)
  - Reference Entities (stories, campaigns, donors via multi-select)
  - Human Approver (assign to admin)
  - Due Date / Priority
- **Task Log Fields:**
  - Status (Queued, In progress, Awaiting approval, Completed, Archived)
  - Generated Output (stored as rich text with diff view)
  - Approval Notes and history
  - Spawned Agent References (future expansion, store JSON metadata)
  - Safety Checklist (checkbox: “Reviewed for PII leakage”, “Meets tone guide”)

### 8. Settings & RBAC
- **User Management Fields:**
  - Name, Email, Role (Admin, Editor, Volunteer)
  - Authentication Provider (Magic Link, Google, Apple)
  - Last Login, MFA status
  - Locale Preference
  - Notification Preferences (webhooks, email summaries)
- **API Keys & Integrations:**
  - Buttondown/Mailchimp API Key store
  - S3/R2 credentials (scoped, encrypted)
  - ReMotion render settings (FPS, fallback image)
  - Feature Flags (homepage redesign enabled, realtime beta, agent lab)
- **Audit Trail:** Table displaying entity, action, user, timestamp, diff summary; export to CSV.

### 9. Data Export & Reports
- **Export Builder:**
  - Dataset selector (Donors, Donations, Campaigns, Stories, Media, Agent Tasks)
  - Filter controls (date range, locale, consent flag)
  - Output format (CSV, JSON)
  - Delivery method (download, email link, S3 presigned URL)
  - Privacy confirmation checkbox prior to export

## Validation & UX Guidelines
- Enforce bilingual completion before publication; display warnings if EN/ES content diverges significantly (±25% word count) using Agent Zero linting.
- Store currency values in cents to avoid rounding issues; display locale-formatted currency in UI.
- Provide autosave drafts every 30 seconds with offline recovery for mobile admins.
- Ensure all forms support keyboard navigation, focus outlines, and ARIA-live regions for validation errors.
- Maintain audit logging for create/update/delete actions with Neon row-level timestamps.
